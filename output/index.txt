}</code></pre><p>　　保存即可，<code>SSH</code> 文件则保持空白。<br>　　拔出 MicroSD 卡，将其插入树莓派。</p><h1>二、配置底层系统</h1><h2>2.1 重新配置用户和密码</h2><p>　　使用 SSH 终端登录至树莓派（默认用户名 <code>pi</code>，密码 <code>raspberry</code>），执行 <code>sudo -i</code> 切换至 root 用户，执行 <code>passwd root</code> 和 <code>passwd pi</code> 修改 <code>root</code> 用户和 <code>pi</code> 用户的密码。</p><p><img src="https://p.itxe.net/images/2021/11/26/imageb1a512ed0d91a793.png" alt="Raspberry-Pi-Proxmox-VE-setup-réinitialiser-passe" title="Raspberry-Pi-Proxmox-VE-setup-réinitialiser-passe"></p><h2>2.2 允许 root 用户通过 SSH 登录</h2><p>　　执行 <code>echo &quot;PermitRootLogin yes&quot; &gt;&gt; /etc/ssh/sshd_config &amp;&amp; echo &quot;PasswordAuthentication yes&quot; &gt;&gt; /etc/ssh/sshd_config</code> 后执行 <code>systemctl restart sshd</code>，使用 root 用户重新 SSH 连接至树莓派即可。</p><h2>2.3 增加和修改源并安装内核头文件</h2><p>　　执行 <code>rm /etc/apt/sources.list.d/*.list</code> 清除不必要的源。<br>　　编辑 <code>/etc/apt/sources.list</code> 文件，将 <code>deb.debian.org</code> 和 <code>security.debian.org</code> 尽数替换为 <code>mirrors.cloud.tencent.com</code> 并在下方添加以下两行内容：</p><pre><code>deb https://raw.githubusercontent.com/pimox/pimox7/master/ dev/
deb http://mirrors.cloud.tencent.com/raspberrypi bullseye main</code></pre><p>　　效果如图</p><p><img src="https://p.itxe.net/images/2021/11/26/imageda5e7405a7070b5b.png" alt="Raspberry-Pi-Proxmox-VE-setup-source-paquet" title="Raspberry-Pi-Proxmox-VE-setup-source-paquet"></p><p>　　其中，前 3 行的 <code>mirrors.cloud.tencent.com</code> 可以替换为其他支持 Debian 11 的源，第 5 行的 <code>mirrors.cloud.tencent.com</code> 可以替换为其他支持 Raspberry Pi OS 的源。部分源并不同时支持两者。</p><p>　　执行 <code>curl https://raw.githubusercontent.com/pimox/pimox7/master/KEY.gpg | sudo apt-key add –</code> 以安装 Pimox7 的 gpg 公钥。</p><p>　　执行 <code>apt update &amp;&amp; sudo apt dist-upgrade -y &amp;&amp; apt install -y raspberrypi-kernel-headers</code> 以更新软件包数据库、更新软件包和安装 raspberrypi 内核头文件。</p><h2>2.4 静态化网卡</h2><p>　　编辑 <code>/etc/network/interfaces</code> 文件，清除原有内容后填入以下内容：</p><pre><code>auto lo
iface lo inet loopback

auto wlan0
iface wlan0 inet static
address 192.168.1.233/24
gateway 192.168.1.1</code></pre><p>　　其中，<code>192.168.1.233/24</code> 和 <code>192.168.1.1</code> 分别为步骤 1.2.1 中提到的树莓派被分配到的 IP 地址和网关。修改完成后保存。</p><h2>2.5 配置主机名</h2><p>　　执行 <code>hostnamectl set-hostname RPi4-PVE-01</code>，其中 <code>RPi4-PVE-01</code> 请自定义。<br>　　编辑 <code>/etc/hosts</code> 文件，清除原有内容后填入以下内容：</p><pre><code>127.0.0.1    localhost
192.168.1.233 RPi4-PVE-01</code></pre><p>　　其中，<code>192.168.1.233</code> 和 <code>RPi4-PVE-01</code> 分别为步骤 1.2.1 中提到的树莓派被分配到的 IP 地址和主机名。</p><h2>2.6 扩大 SWAP</h2><p>　　树莓派默认的 SWAP 大小只有 100M，非常不适合本文场景，故请编辑 <code>/etc/dphys-swapfile</code> 文件，将原先 <code>CONF_SWAPSIZE</code> 的值从 <code>100</code> 改为 <code>2048</code>；执行 <code>/etc/init.d/dphys-swapfile stop &amp;&amp; /etc/init.d/dphys-swapfile start</code> 以应用更改。</p><p><img src="https://p.itxe.net/images/2021/11/26/image58e0f37deeba5c51.png" alt="Raspberry-Pi-Proxmox-VE-setup-augmenter-taille-partition-d-échange" title="Raspberry-Pi-Proxmox-VE-setup-augmenter-taille-partition-d-échange"></p><p>　　一切完成后，重新启动系统。</p><h1>三、安装 Proxmox VE</h1><h2>3.1 安装 Proxmox VE 本体</h2><p>　　将树莓派连接至显示器，插入键盘，执行 <code>apt install -y proxmox-ve</code>。因安装过程中将重置网络，故请勿直接在 SSH 中直接执行 <code>apt install -y proxmox-ve</code>。</p><p>　　弹出配置 Postfix 时，请选择 <code>Local system</code>。</p><p>　　安装需要半个小时左右，建议耐心等待；安装完成后，请重新启动设备。</p><h2>3.2 配置无线网卡和虚拟网桥</h2><p>　　编辑 <code>/etc/network/interfaces</code> 文件，在 <code>source-directory /etc/network/interfaces.d</code> 此行上方插入以下内容：</p><pre><code>auto wlan0
iface wlan0 inet dhcp
        wpa-ssid 114514
        wpa-psk qweasdzxc123

auto vmbr0
iface vmbr0 inet static
        address  10.154.46.1
        netmask  255.255.255.0
        bridge-ports none
        bridge-stp off
        bridge-fd 0
        post-up   iptables -t nat -A POSTROUTING -s '10.154.46.0/24' -o wlan0 -j MASQUERADE
        post-down iptables -t nat -D POSTROUTING -s '10.154.46.0/24' -o wlan0 -j MASQUERADE</code></pre><p>　　其中，<code>wpa-ssid</code> 后为 WLAN 的 SSID，<code>wpa-psk</code> 后为 WLAN 的密码。其他内容请按需修改。<br>执行 <code>systemctl restart networking</code> 以重启网络设备。</p><p><img src="https://p.itxe.net/images/2021/11/27/image.png" alt="Raspberry-Pi-Proxmox-VE-setup-configurer-pont-virtuel" title="Raspberry-Pi-Proxmox-VE-setup-configurer-pont-virtuel"></p><h2>3.3 内核参数调整</h2><p>　　编辑 <code>/etc/sysctl.conf</code> 文件，添加以下内容：</p><pre><code>vm.overcommit_memory=1
net.ipv4.ip_forward=1
net.ipv4.conf.all.rp_filter=1
net.ipv4.icmp_echo_ignore_broadcasts=1
net.ipv4.conf.default.forwarding=1
net.ipv4.conf.default.proxy_arp=0
net.ipv4.conf.default.send_redirects=1
net.ipv4.conf.all.send_redirects=0</code></pre><p>　　保存后执行 <code>sysctl -p</code> 使其生效。</p><p><img src="https://p.itxe.net/images/2021/11/27/image41499ced75a4eafd.png" alt="Raspberry-Pi-Proxmox-VE-setup-réglage-paramètres-noyau" title="Raspberry-Pi-Proxmox-VE-setup-réglage-paramètres-noyau"></p><h2>3.4 配置 Proxmox VE 用户界面</h2><p>　　访问 <code>https://IP 地址:8006</code> （以下以 <a href="https://pve.iks.moe">https://pve.iks.moe</a>:28006 作为演示）登录 Proxmox VE 用户面板。</p><p><img src="https://p.itxe.net/images/2021/11/27/imagefe1a56374b215bba.png" alt="Raspberry-Pi-Proxmox-VE-setup-interface-utilisateur" title="Raspberry-Pi-Proxmox-VE-setup-interface-utilisateur"></p><p>　　用户名和密码与步骤 2.1 中设置的系统用户名和密码一致；语言可选 <code>Chinese (Simplified)</code>。</p><p><img src="https://p.itxe.net/images/2021/11/27/image72846b859e3baefc.png" alt="Raspberry-Pi-Proxmox-VE-setup-certificat-ssl" title="Raspberry-Pi-Proxmox-VE-setup-certificat-ssl"></p><p>　　单击本机唯一的节点，选择“凭证”，单击上方的“上传自定义凭证”按钮上传自定义的证书。此操作会让 Proxmox VE 用户界面在重载时使用自定义的证书，在主机名对应的情况下提供基于 TLS 的身份验证和加密的安全性。（把 <code>pve.iks.moe</code> 解析到步骤 1.2.1 中提到的 <code>192.168.1.233</code> 然后再传证书，刷新即可。）</p><p><img src="https://p.itxe.net/images/2021/11/27/image6b9bac87db4df90f.png" alt="Raspberry-Pi-Proxmox-VE-setup-arriverà-une-fin" title="Raspberry-Pi-Proxmox-VE-setup-arriverà-une-fin"></p><h1>四、创建 QEMU-KVM 虚拟机</h1><h2>4.1 下载镜像</h2><p>　　单击节点下的 <code>local</code>，选择 <code>ISO映像</code>，单击上方的“上传”（如果本地有镜像的话）或“从 URL 下载”下载相应的镜像。</p><p><img src="https://p.itxe.net/images/2021/11/27/imagef06d7f16904b9bd5.png" alt="Raspberry-Pi-Proxmox-VE-setup-télécharger-image-kvm" title="Raspberry-Pi-Proxmox-VE-setup-télécharger-image-kvm"></p><p>　　因树莓派 4B 为 ARM64 架构，因此下载的镜像亦需要为 arm64 架构，腾讯云源 (<a href="https://mirrors.cloud.tencent.com/ubuntu-cdimage/">https://mirrors.cloud.tencent.com/ubuntu-cdimage/</a>, <a href="https://mirrors.cloud.tencent.com/debian-cd/current/arm64/iso-cd/">https://mirrors.cloud.tencent.com/debian-cd/current/arm64/iso-cd/</a> 等有 Ubuntu 和 Debian 的镜像。本章以 Debian 11.1.0 ARM64 为例。</p><p><img src="https://p.itxe.net/images/2021/11/27/image58aabb3bc3544dc8.png" alt="Raspberry-Pi-Proxmox-VE-setup-boîte-dialogue-téléchargement" title="Raspberry-Pi-Proxmox-VE-setup-boîte-dialogue-téléchargement"></p><p>　　将镜像的 URL 粘贴到文本框后单击右边的“查询网址”，待文件名成功显示后即可单击下方的“立即下载”按钮。下载过程将在后台进行，进度将被实时打印。</p><p><img title="Raspberry-Pi-Proxmox-VE-setup-télécharger-journal-kvm"></p><h2>4.2 创建虚拟机</h2><p>　　镜像下载完成后，单击右上角的“创建虚拟机”，弹出相关对话框。</p><p><img src="https://p.itxe.net/images/2021/11/27/imagee6b6c887a2db3ff6.png" alt="Raspberry-Pi-Proxmox-VE-setup-créer-machine-virtuelle" title="Raspberry-Pi-Proxmox-VE-setup-créer-machine-virtuelle"></p><p>　　名称请自定义，其他内容请留空；在单击对话框底部的“下一步”前，请勾选左边的“高级”。</p><p><img src="https://p.itxe.net/images/2021/11/27/imageac80e3dbefea0441.png" alt="Raspberry-Pi-Proxmox-VE-setup-sélection-système-d-exploitation" title="Raspberry-Pi-Proxmox-VE-setup-sélection-système-d-exploitation"></p><p>　　此处“操作系统”选择“不使用任何介质”。</p><p><img src="https://p.itxe.net/images/2021/11/27/imagec4d00d22105eb127.png" alt="Raspberry-Pi-Proxmox-VE-setup-paramètres-virtualisation-couche-matérielle" title="Raspberry-Pi-Proxmox-VE-setup-paramètres-virtualisation-couche-matérielle"></p><p>　　勾选“QEMU代理”，并将 <code>BIOS</code> 改为 <code>OVMF (UEFI)</code>，“添加EFI磁盘”下的“存储”选择 <code>local</code>。</p><p>　　随后的“磁盘大小 (GiB)”和“内存 (MiB)”请按需选择，其他配置建议默认。</p><p><img title="Raspberry-Pi-Proxmox-VE-setup-configuration-terminée-kvm"></p><p>　　单击“完成”以开始创建虚拟机。</p><h2>4.3 调整虚拟机的硬件</h2><h3>4.3.1 删除 CD/DVD 驱动器</h3><p>　　单击刚刚创建的虚拟机，选择“硬件”，单击选中 <code>CD/DVD驱动器 (ide2)</code> 后点击上方的“删除”按钮。</p><p><img src="https://p.itxe.net/images/2021/11/27/image099dd51eec2b1a3d.png" alt="Raspberry-Pi-Proxmox-VE-setup-supprimer-CD-DVD" title="Raspberry-Pi-Proxmox-VE-setup-supprimer-CD-DVD"></p><h3>4.3.2 添加镜像驱动器</h3><p>　　单击上方的“添加”，选择“CD/DVD驱动器”。</p><p><img src="https://p.itxe.net/images/2021/11/27/image933b0661285b3222.png" alt="Raspberry-Pi-Proxmox-VE-setup-créer-CD-DVD-étape" title="Raspberry-Pi-Proxmox-VE-setup-créer-CD-DVD-étape"></p><p>　　“总线/设备” 选择 <code>SCSI</code>，顺序为 <code>2</code>；使用“存储”为 <code>local</code>，“ISO映像”为步骤 4.1 下载的 Debian 11.1 ARM64 镜像；单击对话框底部的“创建“按钮。</p><p><img src="https://p.itxe.net/images/2021/11/27/image66589df713bd2077.png" <code>scsi0</code> 后、在 <code>net0</code> 前的顺序，单击对话框下方的“OK”保存之。</p><p><img src="https://p.itxe.net/images/2021/11/27/image4678bc835c58efb6.png" alt="Raspberry-Pi-Proxmox-VE-setup-ajuster-séquence-démarrage" title="Raspberry-Pi-Proxmox-VE-setup-ajuster-séquence-démarrage"></p><h2>4.4 启动安装</h2><h3>4.4.1 开机</h3><p>　　单击右上角的“启动”按钮，再单击右方的“控制台”进入 noKVM，稍等片刻即可进入 Debian 安装程序。</p><p><img src="https://p.itxe.net/images/2021/11/27/image884c1beda3985ffe.png" alt="Raspberry-Pi-Proxmox-VE-setup-départ-kvm" title="Raspberry-Pi-Proxmox-VE-setup-départ-kvm"></p><p>　　请按照默认值进行安装。</p><h3>4.4.2 手动配置网络</h3><p>　　当出现 <code>Network autoconfiguration fails</code> 时请不要惊慌，因为我们在步骤 3.2 配置的网桥 <code>vmbr0</code> 没有 DHCP 功能，故自动 DHCP 无法生效，此时我们需要指定网络配置。</p><p><img src="https://p.itxe.net/images/2021/11/27/image66dd7e550f63217f.png" alt="Raspberry-Pi-Proxmox-VE-setup-configurer-manuellement-réseau" 虚拟机安装大功告成。</p><h1>五、创建 LXC 虚拟机</h1><h2>5.1 下载镜像</h2><p>　　单击节点下的 <code>local</code>，选择 <code>CT模板</code>，单击上方的“上传”（如果本地有镜像的话）或“从 URL 下载”下载相应的镜像。</p><p><img src="https://p.itxe.net/images/2021/11/27/image11703f36fdead169.png" alt="Raspberry-Pi-Proxmox-VE-setup-télécharger-image-lxc" title="Raspberry-Pi-Proxmox-VE-setup-télécharger-image-lxc"></p><p>　　适用于 ARM64 的 LXC 镜像可以在 <a href="https://uk.lxd.images.canonical.com/images/">https://uk.lxd.images.canonical.com/images/</a> 找到，本章以 Debian 11 为例。</p><p><img URL 粘贴到文本框后单击右边的“查询网址”，待文件名成功显示后修改文件名为方便辨识发行版名称和版本的格式，然后单击下方的“立即下载”按钮。下载过程同样将在后台进行，进度也将被实时打印。</p><p><img src="https://p.itxe.net/images/2021/11/27/image266ce57176edeb82.png" 公钥”（可以通过下方的“加载SSH密钥文件”按钮载入）任选其一设置即可。</p><p><img src="https://p.itxe.net/images/2021/11/27/image6555f98ceb8943a0.png" alt="Raspberry-Pi-Proxmox-VE-setup-modèle-lxc" <code>10.154.46.3/24</code>，“网关 (IPv4)”填写 步骤 3.2 中提到的 <code>10.154.46.1</code>，其他内容保持默认；单击在步骤“确认”下方的“完成”按钮以开始创建 LXC 容器。</p><p><img src="https://p.itxe.net/images/2021/11/27/image8030065fc64a4485.png"
